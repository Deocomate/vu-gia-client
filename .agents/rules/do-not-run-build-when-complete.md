# Do Not Run Build When Complete

To optimize time and resources, **never automatically run build commands** (e.g., `npm run build`, `yarn build`, `pnpm build`, `next build`, or starting the dev server) to test or verify the project after completing a coding task.

- **Skip the build step:** Once you finish implementing the requested changes, stop and do not attempt to compile or build the project yourself.
- **User verification:** The user will manually run the build process and report back with any errors, logs, or feedback if necessary.
- **Action:** Simply notify the user that the code changes have been implemented successfully and wait for their manual verification or further instructions.
